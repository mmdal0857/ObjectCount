# 폰-서버 연결은 유선 우선

폰은 고정 위치에서 상시 전원이 필수이므로, 어차피 존재하는 케이블 하나로 전원과 네트워크를 함께
해결하기로 했다(USB 테더링 또는 USB-C 이더넷 어댑터 + 전원 분배 — 구체 방식은 Phase 1에서 실측
후 확정). Wi-Fi의 간섭·단절·현장별 품질 편차와 공장 IT 협의 부담이 통째로 사라지고, 대역폭이
예측 가능해져 무손실 원칙(ADR-0001)의 재전송 부담도 최소화된다.

## Considered Options

- **전용 AP(5GHz) 패키지**: 배선 자유로우나 무선 고유의 단절·간섭 변수 잔존. 기각.
- **공장 기존 Wi-Fi**: 현장마다 품질·보안정책 복불복, IT 협의 필요. 기각.
- **유선 우선 (선택)**: 안정성·대역폭 최강. 제약은 배선 거리와 서버 포트 수.

## Consequences

- 캡처 앱의 전송 계층은 TCP 기반으로 매체 불문 — 배선이 물리적으로 불가능한 지점은 Wi-Fi를
  예외 경로로 허용한다(같은 프로토콜, store-and-forward가 무선 불안정성을 흡수).
- 설치 패키지에 USB 허브/이더넷 스위치·케이블링이 포함된다. 파일럿 실측 체크리스트에
  배선 거리·경로 항목 추가.
- 폰 프로비저닝이 단순해진다: 유선 링크 위에서 서버 자동 발견(mDNS) + 서버가 기기 시리얼로
  카메라 등록.

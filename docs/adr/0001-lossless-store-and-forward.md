# 무손실 카운팅을 위한 폰 저장 후 전송(store-and-forward)

일반적인 CV 스트리밍은 실시간성을 위해 밀리면 프레임을 버리지만, 이 시스템은 카운트 완전성(보수적
원칙: 조용한 누락 금지)을 실시간성보다 우선하기로 결정했다. 프레임은 폰 로컬 디스크에 먼저 기록되고,
서버가 자기 디스크에 내구성 있게 쓴 뒤 보낸 ack를 받은 분만 폰에서 삭제된다. 이로써 서버 과부하,
Wi-Fi 단절, 서버 재시작이 전부 "폰이 쌓아뒀다가 재전송"이라는 단일 메커니즘으로 복구되고, 카운트
손실은 폰 자체가 고장 난 경우에만 발생한다.

## Considered Options

- **최신 우선 드랍**: 실시간성 최선이나 드랍 = 조용한 누락. 보수적 원칙과 모순되어 기각.
- **서버 디스크 큐만**: 구현 단순하나 Wi-Fi 단절 순간의 프레임을 지키지 못함. 기각.
- **폰 저장 후 전송 + 서버 ack (선택)**: 중고폰의 수~수십 GB 저장공간이 공짜 버퍼가 됨.

## Consequences

- 대시보드의 "실시간" 카운트는 과부하·단절 구간에서 지연될 수 있다 — 대시보드는 카메라별 처리
  지연(lag)을 항상 표시해야 하며, 운영자는 "카운트가 늦게 올라올 뿐 사라지지 않는다"를 전제로 본다.
- 서버 ack는 수신 즉시가 아니라 **디스크에 내구성 있게 쓴 후**에만 보낸다 (수신 중 서버 크래시에도
  무손실 유지).
- 영구 과부하(평균 처리량 < 유입량)는 폰 저장소를 결국 채운다 — 지연·저장소 수위 경보를 두고,
  경보 시 카메라 수 축소나 PC 증설 같은 운영 조치로 해소한다. 시스템이 임의로 버리는 일은 없다.
- 프레임의 무손실 보장은 추론 완료 시점까지다. 추론이 끝난 프레임은 폐기되고, 카운트 이벤트와
  의심 이벤트 스냅샷만 영구 보존된다.

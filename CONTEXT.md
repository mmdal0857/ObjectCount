# ObjectCount

다수의 중고 안드로이드 폰을 카메라로 활용해 컨베이어 위의 등록된 물체를 세는 시스템.
저비용 하드웨어(폰)가 사업의 핵심 경쟁력이며, 지능은 전부 현장 PC에 모인다.

## Language

### 현장 구조

**라인 (Line)**:
카운팅 대상이 되는 컨베이어 한 줄. 항상 정확히 하나의 활성 품종과 하나의 활성 카운트 세션을 가진다.
_Avoid_: 벨트, 레일, 컨베이어(설비 자체를 가리킬 때만 사용)

**품종 (Product)**:
카운팅 대상으로 등록된 물체의 종류. 등록 파이프라인을 통과해야 품종이 된다.
_Avoid_: 오브젝트, 제품, SKU, 아이템

**주 카메라 (Primary Camera)**:
한 라인의 카운트를 결정하는 유일한 카메라. 라인당 정확히 1대.
_Avoid_: 메인 카메라, 대표 카메라

**보조 카메라 (Aux Camera)**:
카운트에 관여하지 않고 다른 각도의 증빙 스냅샷만 남기는 카메라. 라인당 0대 이상.
_Avoid_: 서브 카메라, 예비 카메라(장애 대체용이 아님)

### 카운팅

**카운트 이벤트 (Count Event)**:
물체 1개가 카운트라인을 통과했다는 확정 기록. 집계의 최소 단위.
_Avoid_: 카운트 로그, 통과 기록

**카운트라인 (Count Line)**:
벨트 진행 방향에 수직으로 그은 가상의 선. 트랙 중심점이 이 선을 정방향으로 넘는 순간 카운트가 발생한다.
_Avoid_: 기준선, 트리거라인

**의심 이벤트 (Suspect Event)**:
겹침·면적 이상 등으로 개수 판정이 불확실한 카운트 이벤트. 보수적 원칙에 따라 최소 개수로 세고,
반드시 스냅샷과 함께 재검수 대상으로 남긴다.
_Avoid_: 오류 이벤트, 경고, 이상 감지

**보수적 원칙 (Conservative Counting)**:
개수가 애매하면 적게 센다 — 시스템은 절대 부풀려 세지 않는다. 누락분은 의심 이벤트 재검수로 복구한다.
_Avoid_: 언더카운팅(오차가 아니라 의도된 정책이다)

**무손실 원칙 (Lossless Principle)**:
프레임은 추론이 끝나기 전까지 시스템이 임의로 버리지 않는다 — 카운트는 늦어질 수 있어도
사라지지 않는다. 폰의 저장 후 전송(store-and-forward)으로 구현된다.
_Avoid_: 실시간 보장(정반대의 트레이드오프다)

**미처리 구간 (Backlog)**:
프레임이 폰이나 서버에 쌓여 아직 추론되지 않은 구간. 대시보드에 지연으로 표시되며,
처리되면 카운트가 소급 반영된다.
_Avoid_: 유실 구간, 신뢰불가 구간(손실이 아니라 지연이다)

**카운트 세션 (Count Session)**:
한 라인에서 하나의 품종이 활성화된 연속 구간. 품종 전환(자동) 또는 운영자의 수동 마감으로만
닫힌다 — 시간이 세션을 자르지 않는다. 집계·아카이브의 단위.
_Avoid_: 작업, 배치, 로트

**수동 마감 (Manual Close)**:
운영자가 품종 전환 없이 현재 세션을 닫고 같은 품종으로 새 세션을 여는 행위.
오전/오후 물량 분리, 출하 단위 마감에 쓴다.
_Avoid_: 리셋(카운트가 사라지는 게 아니다)

**보정 (Adjustment)**:
재검수로 확정된 카운트 증감을 담은 append-only 이벤트. 누가 언제 왜 바꿨는지를 항상 지닌다.
원본 카운트 이벤트는 절대 수정되지 않으며, 세션 총카운트는 자동 카운트와 보정의 합산 파생값이다.
_Avoid_: 수정, 정정(원본을 고친다는 오해 유발)

### 등록·배포

**등록 (Registration)**:
새 품종의 샘플 수집 → 라벨링 → 파인튜닝 → 품질 게이트 → 모델 패키지 생성까지의 본사 워크플로우.
_Avoid_: 학습(파인튜닝 단계만을 가리킴), 셋업

**수집 모드 (Collection Mode)**:
캡처 앱이 서버 전송 대신 로컬 녹화로 학습 샘플을 만드는 모드. 현장과 동일한 폰·각도·조명으로
학습 데이터를 확보하기 위해 존재한다.
_Avoid_: 녹화 모드, 학습 모드

**모델 패키지 (Model Package)**:
품종 1개의 가중치 + 메타데이터(품종 ID, 면적 프라이어, 검증 성능)를 묶은 버저닝된 배포 단위.
_Avoid_: 모델 파일, 가중치

### 검증

**리플레이 하네스 (Replay Harness)**:
실제 녹화 영상과 사람이 센 정답 카운트 쌍으로 파이프라인 정확도를 회귀 검사하는 도구.
정확도 개선의 유일한 판정 기준.
_Avoid_: 시뮬레이터(가짜 폰 시뮬레이터는 전송 계층 테스트 도구로 별개)

## Example dialogue

> **개발자**: 3번 라인에서 의심 이벤트가 20건 쌓였는데요.
> **도메인 전문가**: 지금 활성 품종이 뭐죠?
> **개발자**: 오전에 전환해서 현재 세션은 B품종입니다. 대부분 면적 이상 — 두 개가 붙어서 지나간 것 같습니다.
> **도메인 전문가**: 보수적 원칙대로 1개씩으로 세어져 있을 테니, 재검수에서 스냅샷 보고 추가분만 확정하면 됩니다. 보조 카메라 스냅샷도 같이 떠 있나요?
> **개발자**: 네, 측면 각도 증빙이 붙어 있어서 판정은 금방 끝날 겁니다. 확정되면 세션 카운트에 반영되고, 리플레이 하네스 셋에도 이 구간을 추가해 두겠습니다.
